Programação Orientada a Objetos (POO)

A Programação Orientada a Objetos é um paradigma de programação baseado no conceito de "objetos", que podem conter dados e código: dados na forma de campos (atributos) e código na forma de procedimentos (métodos).

Princípios da POO

  • Abstração: Esconder detalhes complexos e mostrar apenas o essencial.
  • Encapsulamento: Proteger os dados, permitindo acesso controlado.
  • Herança: Reutilizar código entre classes.
  • Polimorfismo: Usar a mesma interface para diferentes formas de dados.

Exemplo em Java

public class Pessoa {
    private String nome;

    public Pessoa(String nome) {
        this.nome = nome;
    }

    public void dizerOla() {
        System.out.println("Olá, meu nome é " + nome);
    }
}

Sistemas Embarcados

Sistemas embarcados são sistemas computacionais especializados que fazem parte de um sistema maior, com função dedicada. Eles geralmente operam com recursos limitados de hardware e software, sendo muito usados em automóveis, eletrodomésticos, equipamentos médicos, e dispositivos IoT.

Características

  • Baixo consumo de energia.
  • Alta confiabilidade e tempo real.
  • Limitado em memória e processamento.
  • Costuma utilizar microcontroladores (ex: Arduino, PIC, ESP32).

Exemplo em C para LED com Arduino

/* Configura o pino 13 para controlar um LED */
void setup() {
    pinMode(13, OUTPUT); // configura o pino 13 como saída
}

/* Loop que acende e apaga o LED a cada segundo */
void loop() {
    digitalWrite(13, HIGH); // acende o LED
    delay(1000);            // espera 1 segundo
    digitalWrite(13, LOW);  // apaga o LED
    delay(1000);            // espera 1 segundo
}
